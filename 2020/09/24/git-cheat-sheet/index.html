git cheat sheet

Posted on 2020-09-24 Edited on 2020-09-26 In git

git 常用命令速查

  1. 在线学习git技巧
  2. fetch pull push 的参数可以用 : 表示源和目标
  3. 引用/游标最终指向的是commit，与缓存区工作区无关
  4. 只要有引用(HEAD引用和分支引用)在，它的时间线(log)就不会消失。
  5. 仓库清洗/清理没有引用的 commit（dangling object）:

<rev> : 修订版本 revision，指 commit 的版本ID
<rev>~ 等于 <rev>~1 表示父提交
HEAD~~~ 等于 HEAD~3
<rev>^n 用来表示该提交的第 n 个父提交
单时间线时 HEAD~~~ 等于 HEAD~3 等于 HEAD^^^ 等于 HEAD^1^1^1
git rev-parse HEAD~^2 算出 ID

撤销操作

  1. commit 时间线级别

git reset
–soft：暂存区和工作区都未被改变。
–mixed：暂存区被更新，工作区未被更改。【默认选项】（建议同上)
–hard：暂存区和工作区都将回退。
技巧：git reset --mixed HEAD 撤销暂存区变动，和HEAD保持一致，工作区不变。

reset:移动HEAD以及与HEAD关联的分支引用，未关联时只移动HEAD。
checkout:只移动HEAD，并更新工作区。目标是分支引用时会进行关联，也就是所谓更换分支。更换前强制要求放弃或提交当前的更改。

git revert [HEAD] [HEAD～] ... 连续新增多个相反的 commit 抵消代码更改

  1. 文件级别

git reset HEAD~ [file] 将暂存区中的[file]重置为上一个commit中的[file] (上述三个标签不起作用，只更新暂存区，不更新工作区)
git checkout HEAD~ [file] 将上一个commit中的[file]取出，覆盖缓存区和工作区，使该文件三方一致。
git checkout -- [file] 优先从缓存区取回，如果没有缓存就从仓库取回
猜测checkout后第一个参数表示从哪里取出，–表示暂存区
git checkout HEAD -- [file] 将file从当前commit(HEAD)取出覆盖缓存区(–)和工作区
checkout 可以同时签出多个文件或文件夹
